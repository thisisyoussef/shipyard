# Constitution Check

- [x] New modules respect SRP and dependency direction. `ActivityFeed.tsx` owns feed-level concerns (virtual scrolling, auto-scroll, turn collapse state). `TurnCard.tsx` owns turn-level rendering (header, tool call list, collapsed summary). `ToolCallRow.tsx` owns individual tool call rendering (glance view, expand/collapse, detail view). Each component depends only on the token system (S01), primitives, and view-model types — not on the shell, composer, or other content components. Data transformation remains in `activity-diff.ts` where it belongs.
- [x] Testing strategy covers the change. Unit tests cover each component's rendering and interaction behavior. The `useAutoScroll` hook has dedicated tests. Performance is validated by rendering 200 items and measuring frame rate. Skill-based validation covers spatial quality, visual hierarchy, and overall design. Integration verifies the feed works with real TurnViewModel data from the existing data layer.
- [x] Backwards compatibility respected. The extraction from ShipyardWorkbench preserves the existing data contract: `TurnViewModel[]` and `FileEventViewModel[]` flow from App.tsx unchanged. The new components consume the same view-model types and the same `activity-diff.ts` utility functions. No changes to the WebSocket protocol or backend contracts.
- [x] New dependency justified and risk-assessed. The preferred approach uses no new dependencies (custom virtualizer with IntersectionObserver + ResizeObserver). If the custom approach exceeds the time budget, `@tanstack/react-virtual` (~7KB gzipped) is the fallback — a well-maintained, tree-shakeable library with no transitive dependencies. The inline SVG icon approach avoids icon library dependencies.
